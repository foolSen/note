# Vue 面试

# 双向绑定数据原理

  1. 采用数据劫持结合发布者-阅读者模式的方式
  2. 通过Object,defineProperty() 来劫持各个属性的setter ，getter
  3. 在数据发生变动时发布信息给阅读者，触发相应监听回调

# 单向数据流和双向数据绑定

  • 单向数据流：数据流向是单向的。
    • 优：数据流动的方向可以跟踪，流向单一，追查问题时可以更加快捷。
    • 缺：写起来不太方便，要使UI发生变化需要创建各种 方法 来维护 对应的 属性
    • 例： 父级 prop 的更新会向下流动到子组件中，但是反过来则不行
  • 双向数据绑定：数据之间是相通的，将数据变更的操作隐藏在框架内部
    • 优：表单交互较多的场景下，会简化大量与业务无关的代码
    • 缺：无法追踪局部状态的变化，增加了出错时 debug 的难度

# Vue 生命周期的理解

  • Vue 实例从开始创建、初始化数据、编译模板、挂载Dom→渲染、更新→渲染、卸载等一系列过程，我们称这是 Vue 的生命周期，各个阶段有相对应的事件钩子
  • 生命周期钩子函数
    1. beforeCreate()
    2. created()
    3. beforeMount()
    4. mounted()
    5. beforeupdate()
    6. updated()
    7. beforeDestroy()
    8. destroyed()
    9. vm.$nextTick

# v-if 和 v-show 区别

  • 同： v-if 和 v-show 都能够对元素进行显示与隐藏
  • 异：
    • v-if 是通过 DOM 的创建和销毁这样一个过程
    • v-show 仅仅是通过 CSS 的display:none --- display:block

# $route 和 $router 的路由区别

  • router 是 Vue 的一个实例
  • route 是一个跳转路由的对象，当前的路由信息

# NextTick 是做什么的

  1. 将回调延迟到下次 DOM 更新循环之后执行。
  2. 在修改数据之后立即使用它，然后等待 DOM 更新。

# Vue 组件 data 为什么必须函数

  • 一个组件组件可能存在多个实例

  • 若 data 是 一个对象，对象属于引用类型，则该组件的所有实例共享data对象的所有属性

  • 当 data 是 一个函数时， 函数具有独立作用域快，则每个实例都有自己的作用域空间

# computed 和 methods 的区别

  • 同：都能返回一个值
  • 异：
    • computed：基于它们的依赖属性进行缓存的，当依赖发生改变时，重新求值。
    • method：只要被触发重新渲染， method 调用总会执行该函数。

# 对比 JQuery ，Vue 有什么不同

  • jQuery 专注视图层，通过操作 DOM 去实现页面的一些逻辑渲染
  • Vue 专注于数据层，通过数据的双向绑定，最终表现在 DOM 层面，减少了 DOM 操作
